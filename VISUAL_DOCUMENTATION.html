Inventory Upload
Route: /inventory

Bulk-upload inventory transaction records from a CSV file directly into Oracle Fusion. The system validates each row, streams them to Oracle, and tracks success/failure in real time.

Step-by-step workflow
  1. Click ⬇️ Download Template (top-right) to get the pre-formatted CSV.
  2. Fill the CSV with your inventory data and save it.
  3. Type the Organization Name (e.g. Vision Operations) in the text field.
  4. Drag & drop or click the dropzone to select your CSV file.
  5. Click 📤 Upload & Process — a progress panel appears while Oracle processes each record.
  6. When done, review the result summary. If there are failures, click "View failure details →".

Inventory Template Generation
Route: /inventory-template

Converts a raw Amro inventory export CSV into a correctly formatted Oracle inventory transaction CSV — inverting quantities, deriving transaction types, and summing lines per Branch + Barcode + Order Ref.

Step-by-step workflow
  1. Export the inventory data from the Amro system as CSV.
  2. Drag & drop the file onto the dropzone.
  3. Click 👁️ Preview conversion to see the first 50 converted rows immediately — no Oracle call yet.
  4. Review warnings (skipped rows with missing data appear in a yellow box).
  5. Click ⬇️ Download template CSV to save the converted file.
  6. Use the downloaded CSV as input for the Inventory Upload page.
Mapping rules
  • Branch/Name (text before "/") → SubinventoryCode
  • Product/Barcode → ItemNumber
  • Order Ref → TransactionReference
  • Total (inverted) → TransactionQuantity
  • Base UoM → UOM (default: Each)
Derived values
  • Qty inverted: positive → negative, negative → positive (REFUND keeps positive)
  • Type: Vend Sales Issue when qty < 0; Vendor RMA when qty > 0
  • Date: uses Order Ref date if present, else today

📂 Section 3 – Receipts
🧾
Misc Receipt
Route: /misc-receipt

Upload a CSV of miscellaneous (non-trade) receipts and send each one to Oracle Fusion via SOAP XML. The page supports XML preview before sending, real-time progress tracking, and a detailed response log.

Step-by-step workflow
  1. Click ⬇️ Download Template to get the required CSV structure.
  2. Fill in your misc receipt data (amounts, dates, bank account numbers, etc.).
  3. Drop the CSV onto the dropzone.
  4. (Optional) Click 👁️ Preview XML to review the SOAP payloads that will be sent — useful for debugging.
  5. Click 📤 Upload & Send to Oracle to begin submission.
  6. Watch the live progress panel — it shows Total / Success / Failed counts as each receipt is processed.
  7. After completion, expand View Detailed Response Log to see Oracle's raw responses.
SOAP XML preview example
<soapenv:Envelope xmlns:soapenv="..."> <soapenv:Body> <tns:createMiscellaneousReceipt> <ReceiptNumber>MISC-2025-001</ReceiptNumber> <Amount>5000.00</Amount> <CurrencyCode>SAR</CurrencyCode> <ReceiptDate>2025/05/01</ReceiptDate> <BankAccountNumber>SA12345678</BankAccountNumber> </tns:createMiscellaneousReceipt> </soapenv:Body> </soapenv:Envelope>

Standard Receipt
Route: /standard-receipt

Upload customer cash/payment receipts from a CSV and submit them to Oracle Fusion via REST API. Supports payload preview (JSON), real-time progress, and Oracle response logging.

Step-by-step workflow
  1. Click ⬇️ Download Template and prepare your standard receipt CSV.
  2. Drop or select the CSV file.
  3. (Optional) Click 👁️ Preview Payload to review the JSON that will be POSTed to Oracle.
  4. Click 📤 Upload & Send to Oracle.
  5. Monitor the progress panel and expand the response log if needed.
  6. Click View Details in the history table to see row-by-row results.

🔗
Apply Receipt
Route: /apply-receipt

Links existing Oracle receipts to their corresponding invoices by uploading a CSV that maps InvoiceNumber → ReceiptNumber(s). Sends SOAP requests to Oracle and tracks each application with live progress.

Step-by-step workflow
  1. Click ⬇️ Download Template and fill in invoice-to-receipt mappings.
  2. Drop or select the CSV file.
  3. (Optional) Click 👁️ Preview Payload to review what will be sent to Oracle.
  4. (Optional) Click ✅ Verify Receipts to validate that every receipt exists in Oracle before applying.
  5. Click 📤 Upload & Send to Oracle to apply all receipts.
  6. Monitor the progress panel; expand response log for Oracle feedback.
CSV structure
InvoiceNumber ReceiptNumber1 ReceiptNumber2 ReceiptNumber3 …
💡 Tip: You can have unlimited ReceiptNumberN columns — the system detects them all automatically. One invoice may be linked to multiple receipts.

AR Invoice Upload (Creation)
Route: /ar-invoice

Create a single AR Invoice in Oracle Fusion by pasting a JSON payload directly. Also supports bulk submission of multiple payloads generated from the AR Invoice Data page or the Vend Invoice page.

Single invoice – step-by-step
  1. Click 📋 Load Sample to populate a pre-filled example payload.
  2. Edit the JSON in the textarea: set BusinessUnit, BillTo fields, invoice lines, etc.
  3. Click ✨ Format JSON to auto-indent the payload for readability.
  4. Click 🚀 Create AR Invoice — Oracle's response (including the new Invoice Number) appears below.
  5. Click View Full Details → to see the stored response record.

AR Invoice Data
Route: /ar-invoice-data

A staging area for AR invoice line items. Upload invoice data from CSV into the database, review it, then auto-generate Oracle-ready JSON payloads — without sending anything to Oracle yet. Use this when you want to stage and review data before committing.

Step-by-step workflow
  1. Click 📥 Download Template and fill in invoice line items.
  2. Drop or select the CSV file and click 📤 Upload CSV — records are stored in the database as a batch.
  3. The new batch appears in the Uploaded Batches table. Click View Records to inspect all lines.
  4. Click Generate Payload next to a batch — the system groups lines by customer into one invoice per customer.
  5. Review the generated JSON payloads. Click 📋 Copy JSON or click → Use this payload in AR Invoice Creation to send it to Oracle.
  6. To clean up, click Delete on any batch you no longer need.
Required CSV columns
customerName itemNumber description quantity unitSellingPrice taxClassificationCode transactionDate accountingDate paymentTerms invoiceCurrencyCode
Optional CSV columns
customerNumber siteNumber subinventory businessUnit transactionSource crossReference salesOrder memoLine
💡 If customerName + subinventory are provided, header fields are auto-populated from the FusionSalesMetadata table.

AR Invoice Response
Route: /ar-invoice-response

A read-only viewer for all AR invoice responses stored from Oracle Fusion. Displays two linked tables: Invoice Headers and Invoice Lines. Click any header row to filter lines by that specific invoice.

Step-by-step usage
  1. Open the page — both tables load automatically with pagination (20 headers / 50 lines per page).
  2. Scan the AR Invoice Header table: check Status, Transaction Number, Bill-To Customer, and Oracle message.
  3. Click any header row to filter the AR Invoice Lines table to only that invoice's lines.
  4. Click Clear filter ✕ to show all lines again.
  5. Use ← Previous / Next → buttons to paginate through large datasets.

📂 Section 5 – Vend

Vend Invoice
Route: /vend-invoice

Converts two Vend Excel exports (Payment Lines + Sales Lines) into Oracle AR Invoice payloads, automatically grouped by store × date × payment type. Supports TABBY and TAMARA as separate invoice streams.

Step-by-step workflow
  1. Export Payment Lines and Sales Lines from your Vend system as Excel files (.xlsx).
  2. Drag & drop (or browse) Payment Lines into the left upload box.
  3. Drag & drop (or browse) Sales Lines into the right upload box.
  4. Click 📤 Generate AR Invoice Payloads — the backend groups data by store and date.
  5. Review the generated payloads in the result panel below.
  6. Choose an action:
    • 📥 Download JSON — saves all payloads as a timestamped .json file.
    • 📥 Download CSV — saves a summary CSV for record-keeping.
    • 🚀 Send to AR Invoice — transfers all payloads to the AR Invoice Upload page for bulk submission to Oracle.

Logs & Failures
Route: /failures/:uploadId

Shows the row-level failure details for a specific upload. Each failed row is listed with its row number and the exact Oracle error message. Accessible from the sidebar (shows failures across all uploads) or from a specific "Failures" link in any upload history table.

How to use
  1. Click the sidebar ⚠️ Logs & Failures link to open the global failure view (uploadId = 0).
  2. Alternatively, from an upload row in Inventory Upload or Reports → Upload History, click the Failures link to jump directly to that upload's failures.
  3. Review each row: the Row # column tells you exactly which CSV row failed.
  4. Read the Error column — Oracle errors like "AR-855636: item not found" tell you the root cause.
  5. Fix your CSV data for those specific rows and re-upload.

User Management ADMIN ONLY
Route: /admin/users

Manage all portal user accounts. Available only to Admins — the sidebar item is hidden for regular users and managers. Supports create, edit, enable/disable, and forced password reset.

Available actions
  • + Add User — opens a modal to create a new account with email, password, and role (User / Manager / Admin).
  • Edit — modify an existing user's email, role, or active status.
  • Disable / Enable — toggle a user's access without deleting the account. Disabled users cannot log in.
  • Reset PW — generates and displays a new temporary password for the user.
